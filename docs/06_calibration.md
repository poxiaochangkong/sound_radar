# 06 — 标定流程

## 1. 目的

声音雷达默认参数已对多数 FPS 游戏可用,但在不同环境(安静房间 vs 嘈杂)和不同游戏(混响不同)下可能需要微调。本文档说明:
1. 软件内置的实时灵敏度滑杆——日常使用
2. 可选的 `tools/calibrate.py`——生成针对特定游戏/环境的 profile

**重要**:核心代码**永不依赖** profile。profile 只覆盖默认配置,缺失时软件仍正常运行。

---

## 2. 快速标定:实时灵敏度滑杆

UI 提供单个 `Sensitivity` 滑杆(范围 0–100),背后映射到核心参数:

```
slider_value (0..100)
    │
    ▼
sensitivity_factor (0.5..2.0)   # 线性映射
    │
    ├──▶ effective_snr_threshold = snr_threshold_db / sensitivity_factor
    └──▶ effective_flux_multiplier = flux_multiplier / sensitivity_factor
```

| 滑杆值 | 含义 | 行为 |
|---|---|---|
| 50(默认) | 标准 | 适合多数游戏 |
| < 50 | 更敏感 | 安静环境、远距离脚步;误报增多 |
| > 50 | 更不敏感 | 嘈杂环境、爆炸频繁;漏报增多 |

**调参建议**:
1. 进入游戏空旷处,无人走动 → 调到刚好**无任何雷达点**(通常 55–70)
2. 让朋友在 10 米外走动 → 调到能**稳定显示**其方位(通常 40–55)
3. 长时间游戏后微调

## 3. 高级标定:`tools/calibrate.py`

### 3.1 适用场景
- 默认参数在某款游戏上效果差
- 想生成可分享、可复用的 profile
- 想量化本底噪声绝对值

### 3.2 标定流程(全自动)

```
步骤 1:启动 calibrate.py(带游戏名参数)
   │
   ▼
步骤 2:[5 秒安静段] 提示"请保持游戏内无人走动"
   │   采集 5 秒多声道 PCM
   │   计算每声道平均 RMS² → noise_floor_baseline
   │
   ▼
步骤 3:[10 秒活动段] 提示"请正常游戏:走动、开枪、爆炸"
   │   采集 10 秒多声道 PCM
   │   计算每声道能量分布直方图
   │   计算 spectral flux 时间序列
   │
   ▼
步骤 4:自动生成参数
   │   snr_threshold_db = 10 * log10(P95_activity / median_quiet)
   │   flux_multiplier  = 2.5  (自适应,几乎不需调)
   │   decay_ms         = 根据事件间隔中位数设置
   │
   ▼
步骤 5:写入 profiles/<game>.yaml
```

### 3.3 输出 profile 格式(`profiles/<game>.yaml`)
```yaml
# Auto-generated by tools/calibrate.py
game: csgo
generated_at: 2024-01-01T12:00:00
environment:
  noise_floor_baseline:
    L: 1.2e-05
    R: 1.1e-05
    # ... per channel
recommended:
  direction:
    snr_threshold_db: 14.3
  onset:
    flux_multiplier: 2.8
  tracking:
    decay_ms: 350
```

### 3.4 使用 profile
启动时通过命令行参数指定:
```powershell
python src/main.py --profile profiles/csgo.yaml
```
软件会**合并** profile 到默认 config(profile 优先),其余参数不变。

## 4. 多声道布局自检

启动时软件会自动检查:
- WASAPI 实际返回的声道数
- 与 `config.yaml` 中 `channel_layout` 是否匹配
- 不匹配时:**报错并引导**,绝不静默下混

```
[ERROR] Configured channel_layout=7.1 (8 channels),
        but WASAPI returns 2 channels.
        Fix: set Windows default device to 7.1 surround.
```

## 5. 虚拟声卡(VoiceMeeter)配置说明

### 5.1 为什么需要?
让游戏输出 7.1,同时让玩家耳机仍能听到方位化音频(通过系统 HRTF 虚拟化)。

### 5.2 配置步骤
1. 安装 VoiceMeeter Banana(免费)
2. Windows 声音设置:默认输出设备设为 "VoiceMeeter Input"
3. VoiceMeeter 设置:
   - Hardware Out = 你的物理耳机
   - 在 Hardware Out 上启用 A1 → WDM → 选择耳机
   - 启用 HRTF 虚拟化(右键 A1 → 耳机图标)
4. VoiceMeeter Input 声道配置:7.1
5. 游戏内:扬声器模式选 7.1,HRTF 关闭

### 5.3 验证
启动 `tools/record_session.py --seconds 5`,看录到的 wav 是不是 8 声道。

## 6. 故障排查

| 症状 | 可能原因 | 检查 |
|---|---|---|
| 雷达完全无反应 | 设备不是 7.1 | WASAPI 声道数自检 |
| 雷达永远在正前 | C 声道混入 UI 声 | `ignore_center_channel: true` |
| 方位明显反相(左右反) | 声道顺序与标准不符 | 用 record 工具检查每声道 |
| 枪声无反应 | flux_multiplier 太高 | 滑杆调小 |
| 脚步过于敏感 | snr_threshold 太低 | 滑杆调大 |
